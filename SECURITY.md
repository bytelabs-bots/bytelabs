# Security Policy

## Supported Versions

Security updates are applied to the latest version of this project in the active repository.

## Reporting a Vulnerability

Please do not open a public issue for security concerns.

Instead, report them privately by emailing the maintainer or opening a private security report through the repository's security contact.

Please include:
- a clear description of the issue
- steps to reproduce it
- the affected files or endpoints
- any suggested mitigation

## Response Expectations

- We will acknowledge reports as soon as possible.
- We will investigate and provide an update before making any public disclosure.
- If the issue is confirmed, we will coordinate a fix and release timeline.

## Security Best Practices

- Never share Discord tokens or vault secrets in issues, pull requests, or chat.
- Keep local environment variables and credentials private.
- Use strong master passwords and avoid committing sensitive data.
- Review dependency updates regularly.
